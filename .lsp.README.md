# LSP Configuration

The `.lsp.json` file configures which language servers are **available** to the plugin.
The LSP **built-in tool** then uses these servers for code intelligence.

## How It Works

1. `.lsp.json` declares language servers (e.g., `typescript-language-server`)
2. The LSP built-in tool auto-loads configured servers when the plugin is active
3. After every `Edit`/`Write` operation, the LSP tool auto-reports type errors
4. The agent gets real-time type feedback without manual `tsc --noEmit` runs

## Supported Operations

- **diagnostics** — type error reporting (automatic after file edits)
- **definition** — jump to definition
- **references** — find all references
- **typeInfo** — get type information at cursor position
- **symbols** — list document/workspace symbols
- **implementations** — find implementations of interfaces/abstract classes
- **callHierarchy** — trace incoming/outgoing call hierarchies

## Prerequisites

Install the TypeScript language server globally:

```bash
npm i -g typescript-language-server typescript
```

## Important

The LSP tool is a **built-in Claude Code tool**, not just a config file. The `.lsp.json`
tells the plugin which servers exist; the tool provides the actual code intelligence
capabilities used during code review, refactoring, and editing tasks.
